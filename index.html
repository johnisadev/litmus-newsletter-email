Ghost Tables To tackle Rendering On Outlook

The term "ghost" comes from the idea that tables only get rendered on Outlook. Check out this blog to learn more about how HTML tables are required for consistent personalization on Outlook.

  • role="presentation" For emails we use tables to create layouts but it's a work-a-round. This can create accessability issues with screen readers. It's important to add "presentation" for the "role" attribute

  • Semantic markup should be used as much as possible. h1 to h6 tags should be used for headings and p tags for paragraphs. Container tags such as header and footer should be used with caution as they are not supported by several major email clients.

  • Margin or padding should be the default methods for adding whitespace. multiple empty td tags and multiple br tags should be avoided.
